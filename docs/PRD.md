## 1. 개요

- **한 줄 정의:** 해외 혼행자를 위해, 오늘의 일정·무드에 맞는 동행을 짧고 가볍게 매칭하고 안전한 정산까지 한 흐름으로 해결하는 반응형 웹 서비스
- **해결하는 문제:** 혼자 여행 중 즉흥적으로 "오늘 저녁 같이 스테이크 먹을 사람", "내일 디즈니랜드 같이 갈 사람"처럼 짧은 단위의 동행을 빠르게 구하기 어렵고, 낯선 사람에 대한 신뢰 불안과 비용 정산 문제가 혼행의 경험을 제한한다
- **타깃 사용자:** 유럽 등 해외를 혼자 여행 중인 한국인 여행자 (특히 즉흥적으로 식사·투어·전시·놀이공원 등 특정 활동의 동행을 찾고 싶은 사람)
- **플랫폼:** 반응형 웹, 모바일 우선 (max-width 420px)

---

## 2. 배경 & 근거

- **시장/경쟁 맥락:** GAFFL·NomadHer·TripBFF 등 여행 동행 서비스가 존재하지만 대부분 여행 전체 메이트 탐색이나 커뮤니티 중심이다. 비용 보관·정산 안전장치가 없고, 오늘/내일 단위의 초단기 즉흥 매칭에 특화된 서비스는 비어 있다
- **우리의 차별 프레이밍:** TRADDY는 해외 혼행자를 위한 `same-day travel companion layer`다. 일정·루트·무드 기반 추천, 조건형 모집, 후기/인증, 에스크로형 엔빵 정산을 한 흐름으로 묶어 "오늘 저녁/내일 오전" 단위의 가볍고 안전한 동행 경험을 제공한다

---

## 3. 목표 (Goals)

1. 혼행자가 복잡한 절차 없이 3분 안에 오늘의 동행 후보를 발견하는 경험을 만든다
2. 본인 인증·여행 성향 프로필·후기를 통해 낯선 사람에 대한 신뢰 불안을 실질적으로 줄인다
3. 에스크로형 정산으로 비용 관련 분쟁 없이 동행이 자연스럽게 마무리되도록 한다

## 4. User Scenarios   ← userflow

### Scenario 1 — 오늘의 동행 발견
- **사용자:** 파리에서 혼자 여행 중인 한국인 여행자
- **상황:** 오늘 저녁 오르세 미술관에 혼자 가기 애매해서 같은 취향의 동행을 빠르게 찾고 싶은 상황
- **목표:** 나의 오늘 일정·무드와 맞는 동행 후보를 빠르게 발견하고 프로필을 확인한다
- **흐름:**
  1. 환영 화면에서 [내 일정으로 추천받기] 탭
  2. 빠른 매칭 설정에서 지역(마레), 시간대(오늘 저녁), 활동(전시) 선택 후 [추천 받기]
  3. 추천 홈에서 인증 완료된 사람 카드와 모집글 카드 확인
  4. 관심 있는 카드 탭 → 동행 상세에서 프로필·후기·조건 확인
  5. [합류 요청하기] 또는 [찜하기]
- **완료 조건:** 추천 홈에서 동행 후보 카드가 1개 이상 표시되고, 사용자가 상세 화면까지 진입한다

### Scenario 2 — 동행 모집글 작성
- **사용자:** 스테이크 맛집 예약을 혼자 가기 아까워 1명의 동행을 구하고 싶은 여행자
- **상황:** 내가 먼저 모집글을 올려 조건에 맞는 사람을 기다리고 싶은 상황
- **목표:** 시간대·활동·비용·인원·성별 조건이 담긴 짧은 단위 모집글을 작성하고 게시한다
- **흐름:**
  1. 모집글 작성 화면 진입
  2. 활동 유형, 시간대, 장소, 모집 인원, 성별 조건, 예상 비용 입력
  3. 게시 → 추천 홈 또는 탐색 피드에 노출
- **완료 조건:** 모집글이 게시되어 다른 사용자의 추천 홈에 노출 가능한 상태가 된다

### Scenario 3 — 합류·정산·후기
- **사용자:** 모집글에서 합류 요청을 한 여행자
- **상황:** 동행 신청이 수락되어 같이 움직이고, 일정 종료 후 비용 정산과 후기를 마무리하고 싶은 상황
- **목표:** 비용을 안전하게 정산하고 상대방에게 후기를 남긴다
- **흐름:**
  1. 합류 요청 → 수락 → (필요시) 참가비 에스크로 보관
  2. 일정 진행
  3. 일정 완료 확인 → 정산 완료 → 후기 작성
- **완료 조건:** 정산이 완료 처리되고 양측 후기가 각자의 프로필에 반영된다

## 6. Success Criteria

- SC-001: 신규 사용자가 서비스 최초 진입 후 3분 이내에 추천 동행 후보 화면(추천 홈)에 도달할 수 있다
- SC-002: 사용자가 동행 상세 화면의 프로필·후기·조건 정보만으로 합류 여부를 판단할 수 있다 (합류 전 별도 채팅 없이 판단 가능)
- SC-003: 비용이 발생한 동행의 정산이 서비스 내 에스크로를 통해 완료되어 미정산 분쟁이 발생하지 않는다
- SC-004: 모집글 작성자가 별도 설명 없이도 조건 설정 화면을 완료할 수 있다 (첫 사용 기준)

## 7. Edge Cases

- **추천 결과 없음:** 설정한 지역·활동·무드에 매칭되는 후보가 없을 때 → "아직 딱 맞는 동행은 안 보이네요" 상태와 함께 비슷한 루트 탐색 CTA 제공 (빈 상태 UX)
- **인원 마감 후 신청:** 이미 모집 완료된 글에 신청 시 → "이미 마감된 모집이에요" 안내 후 유사 모집글 추천
- **중복 신청:** 동일 모집글에 재신청 시도 → 이미 신청 중 상태임을 표시하고 중복 요청 차단
- **합류 요청 취소/이탈:** 수락 전 취소 시 → 정상 취소 처리 / 수락 후 에스크로 입금 완료 상태에서 취소 시 → 환불 처리 정책 적용 (정책 미정 → 오픈 이슈)
- **본인 인증 미완료:** 인증 없이 탐색은 허용 → 합류 요청 시점에 인증 요구 (가입/인증은 마지막 단계까지 미루어 초기 마찰 최소화)
- **모집글 작성 중 이탈:** 작성 중인 내용 임시 저장 또는 이탈 확인 다이얼로그 표시
- **정산 완료 불일치:** 양측의 일정 완료 확인이 불일치할 경우 → 운영팀 개입 프로세스 필요 (프로세스 미정 → 오픈 이슈)
- **네트워크 오류:** API 호출 실패 시 → "잠시 연결이 흔들렸어요. 한 번만 다시 불러올게요." 안내 후 재시도 유도
- **위치 정보 미제공:** GPS 접근 거부 시 → 사용자가 직접 지역 선택(텍스트 입력 또는 도시 선택)으로 대체

## 9. 브랜드 & 디자인   ← brandvoice + DESIGN

- **서비스 이름:** TRADDY
- **한 줄 무드:** 낯선 도시에서도 안심을 먼저 건네고, 함께할 가능성을 설레게 열어주는 든든한 여행 메이트
- **보이스 요약:** 낯선 도시에서 먼저 긴장을 풀어주고, 혼자만의 여행도 존중하면서 새로운 사람과의 하루를 부드럽게 연결해주는 여행 메이트. 안심과 설렘 사이의 균형을 잃지 않는다
- **UI 카피 톤 가이드:**
  - Do: 제안형("찾아볼까요?", "함께일 수도 있어요"), 혼자 여행 존중, 빈 상태에서도 안심 먼저
  - Don't: 조급함 유도("놓쳐요", "지금 당장"), 소개팅앱 뉘앙스, 과잉 감성 문구
  - 존댓말 기본("~해요", "~볼까요?"), 짧고 쉬운 문장, 느낌표 제한적 사용
- **시각 디자인 시스템:** → docs/DESIGN.md 참조 (색·타이포·간격·컴포넌트)
  - Primary: `#eb2b51` (CTA), Canvas: `#ffffff`, Accent-Mint: `#1ec7be` (신뢰·탐색)
  - 폰트: Pretendard, 모바일 420px max-width

## 10. 범위 / 비범위 & 우선순위

- **v1 범위:**
  - 빠른 매칭 설정 + 추천 홈 (지역·활동·무드 기반)
  - 동행 상세 (프로필·성향 태그·인증 배지·후기)
  - 모집글 작성 (활동·시간대·인원·성별 조건)
  - 합류 요청 + 수락/거절
  - 본인 인증 (합류 요청 전 필수)
  - 에스크로형 정산 (참가비 있는 동행)
  - 동행 후기 작성

- **Out of scope (지금 안 함):**
  - TRADDY 전용 여행 성향 테스트 (P2)
  - 재매칭 제안 기능 (P2)
  - 다국어 지원 (영어·현지어 등)
  - 여행 계획 저장·공유 기능
  - 그룹(3인 이상) 동행 관리 전용 UI

- **우선순위 묶음(제품 관점):**
  - MVP: FR-001, FR-002, FR-003, FR-004, FR-005, FR-006 (추천·상세·모집·인증·합류)
  - v1: MVP + FR-007, FR-008, FR-009, FR-010, FR-011 (정산·후기·찜·프로필·채팅)
  - 이후: FR-012, FR-013 (성향 테스트·재매칭)

## 11. Assumptions

- 서비스 언어는 한국어 단일 제공 (첫 버전은 해외 혼행 한국인 대상, 현지인 대상 아님)
- 첫 버전 대상 국가·도시는 유럽 주요 관광지 기준 (파리, 런던, 바르셀로나 등 한국인 혼행 수요가 높은 도시)
- 위치 정보는 정확한 GPS 좌표를 노출하지 않고 지역/구 단위 텍스트로만 표시 (안전 설계)
- 합류 요청 수락 전까지 가입 없이 탐색·열람이 허용된다 (가입 장벽을 최대한 뒤로 미룸)
- 채팅은 합류 요청 수락 이후에만 오픈된다 (신뢰 판단 → 합류 결정 → 소통 순서)
- 결제 연동은 국내 카드 및 간편결제 기준으로 설계한다
- 에스크로 정산은 비용이 발생하는 동행(투어·입장권 등)에만 적용하고, 자유 합류(무료 동행)는 별도 결제 없이 진행한다
